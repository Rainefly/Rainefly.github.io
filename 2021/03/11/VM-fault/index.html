一次与虚拟机的搏斗——联网

Posted on 2021-03-11 Edited on 2021-03-25

这是个问题

我也不是第一次用虚拟机了，也不是第一次玩xp了。

但是，时隔两个月，我打开了xp的虚拟机，发现怎么都连不上网。

网上有一种介绍xp联网的方式：

  1. 主机的网络邻居属性—>本地连接属性—>”高级”—>’Internet连接共享’–允许其他网络用户通过此计算机的Internet连接来连接：家庭网络连接选（VMware Network Adapter VMnet8）；

  2. 打开虚拟机设置，点击左侧的“网络适配器”，选择为NAT模式；

  3. 点击虚拟机中的“编辑”——“虚拟网络编辑器”，选中“VMnet8”，设为NAT模式；

  4. 右击XP系统的“本地连接”，选择“属性”，再双击“Internet协议(TCP/IP)”，将IP地址和DNS服务器地址都设为“自动获取”，点击“确定”。

（当然，也可以尝试直接从Vmware Station的设置入手，把网络适配器的模式改成NAT，一般都能解决问题。桥接和host-only都容易出问题。）

但是，我发现我的“高级”里面没有“Internet连接共享”这个设置。很好，卡死在了第一步。

还好解决了

我打开了我的另一个linux虚拟机，发现依然连不上网。由此判断应该是本机的问题。

经过排除，最终发现是services.msc里面关于VM的NAT和DHCP服务都没有开起来，虽然已经设置成了自动，但还是没有开启。最终手动开，问题解决。

事后反思，应该是自己习惯在平时开机清理内存占用时，会用一些安全软件关掉VM启动时后台的运行进程。所以，才会使得VM的NAT和DHCP关闭。
